Git and GitHub Workflow Summary

1. Create repository on GitHub

  1. Sign in to GitHub and click + â†’ New repository.
  2. Choose a name (for example, odin-foundations-course-).
  3. Leave it empty (no README) and click Create repository.

2. Clone repository to local machine

  1. Open the terminal on Linux.
  2. Change to your home directory: 
    cd ~
  3. Clone the repository: 
    git clone https://github.com/SEU_USUARIO/odin-foundations-course-.git
cd odin-foundations-course-
git status
git remote -v

3. Add project files

  1. Create a folder for the project and copy the files: 
    mkdir links-and-images
cp -r ~/links-and-images/* links-and-images/
git status
  2. Untracked files are ready to be added to Git.

4. Create the first commit

  1. Stage the files: 
    git add .
git status
  2. Create the commit: 
    git commit
    Use a subject and body, for example: 
    Add initial links and images project

Add the first HTML files and images for the Odin Project exercise.
  3. Check the last commit: 
    git log -1

5. Push commits to GitHub

  1. Send the commits to the remote repository: 
    git push -u origin main
  2. Next time, only: 
    git push

6. Daily workflow for new commits

  1. Edit files in VS Code and save.
  2. Check changes: 
    git status
  3. Stage changes: 
    git add arquivo.html   # or git add .
  4. Create a new commit: 
    git commit
  5. Push to GitHub: 
    git push
